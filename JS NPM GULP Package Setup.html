WORKFLOW from clone Initial setup as long as package.json and bower.json list dependencies - npm install - bower install - gulp build WORKFLOW from scratch PHASE 1 Step 1. npm init Step 2. npm install gulp --save-dev or npm install gulp -g for global Step 3. create gulpfile.js Step 4. npm install browserify --save-dev Step 5. create .gitignore node_modules/ bower_components/ build/ temp/ .DS_Store Step 6. npm install vinyl-source-stream --save-dev Step 7. npm install gulp-concat --save-dev Step 8. npm install gulp-uglify --save-dev Step 9. npm install gulp-util --save-dev Step 10. npm install del --save-dev Step 11. npm install jshint --save-dev and npm install gulp-jshint --save-dev Step 12. npm install bower -g - bower init - bower install jquery --save - bower install bootstrap --save (Step 13. bower install moment --save - For dates and times) Step 14. npm install bower-files --save-dev Step 15. npm install browser-sync --save-dev Step 16. npm install gulp-sass gulp-sourcemaps --save-dev PHASE 2 Step 17. Add Requires to gulpfile Step 18. Add Gulp Tasks -- Detailed Breakdown 1. NPM. Node Package Manager - npm init - will create Package.json which is our manifest that stores our packages - will create node_modules which will be ignored NPM Packages 2. Gulp - npm install gulp --save-dev or for global npm install gulp -g - optimizes our code and packages it up in a format that the browser can understand. It is also in charge or using all the other npm packages - saves package list to package.json - similar to ruby cache. - will need to make a gulpfile.js which will need require and gulp.task code - gulp tasks need to be run in terminal each time your files change as you develop - gulp jsBrowserify 3. Browserify - npm install browserify --save-dev - is responsible for using require and exports keywords to translate the code into JavaScript our browser does understand. - follows the require statements in the interface and collects all the code - Gulp is used to run browserify 4. Gitignore - .gitignore node_modules - Commit the package.json file with the project but not the node_modules node_modules/ bower_components/ build/ temp/ .DS_Store 5. When we clone the project to continue working on it, we simply run npm install and all packages saved to the manifest file will be downloaded into a new node_models/ folder 4. Vinyl-source-stream - npm install vinyl-source-stream --save-dev - used for putting the browserified source code into a new file 5. Gulp - Concat - npm install gulp-concat --save-dev - process of consolidating multiple interface files into one - run gulp jsBrowserify for new version of browserify 6. Minifying with Gulp-uglify - npm install gulp-uglify --save-dev - After require is added to gulpfile/tasks then you need to run gulp minifyScripts 7. Environmental Pro/Dev with Gulp-util - npm install gulp-util --save-dev - Run gulp build --production for production - Run gulp build for developmental 8. Clean Tasks with del - npm install del --save-dev - used to delete tmp and build folder 9. JSHint - linter - npm install jshint --save-dev and - npm install gulp-jshint --save-dev - run gulp jshint to find errors // Front-end to handle dependencies like jquery and bootstrapPing 10. Install Bower - npm install bower -g - bower install for setup from beginning and each dependency - sudo npm install bower -g - bower init - bower install jquery --save - bower install bootstrap --save 11. Moment - bower install moment --save - lets us work with dates and times 12. Bower-files - npm install bower-files --save-dev - gulp bowerJS run everytime we add a new js front end dependency - allows streamline of link and script files // Server 13. BrowserSync - npm install browser-sync --save-dev - local server with live reloading - gulp serve : launches local server //SASS 14.SASS - npm install gulp-sass gulp-sourcemaps --save-dev
